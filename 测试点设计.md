## 十、数据库数据一致性校验

### 10.1 核心表校验清单

| 表名 | 操作 | SQL 校验点 |
|------|------|-----------|
| **user** | 注册 | `SELECT * FROM user WHERE username = '{注册用户名}'` → 校验 nickname / email / phone / role / status / create_time 字段正确写入；password 为 BCrypt 加密串 |
| **user** | 修改密码 | 修改前后 password 字段不同 |
| **user** | 删除用户 | `SELECT deleted FROM user WHERE id = {id}` → deleted = 1（逻辑删除） |
| **skill** | 新增 | `SELECT * FROM skill WHERE name = '{技能名}'` → 校验 category / level / description 一致 |
| **study_record** | 新增 | `SELECT * FROM study_record WHERE user_id = {id}` → 校验 content / duration / type / status |
| **resume** | 创建 | `SELECT * FROM resume WHERE user_id = {id}` → 校验 content / evaluation / score / strengths / suggestions / status |
| **orders** | 新增 | `SELECT * FROM orders WHERE id = {id}` → 校验 status 初始值为 pending |
| **orders** | 更新 | `SELECT status FROM orders WHERE id = {id}` → 校验状态流转正确 |
| **ai_config** | 启停 | `SELECT enabled FROM ai_config WHERE id = {id}` → 校验 true 或 false |

### 10.2 数据库校验 SQL 示例

```sql
-- 1. 校验注册用户
SELECT id, username, nickname, role, create_time, email, phone
FROM user
WHERE username = 'test001';

-- 2. 校验逻辑删除
SELECT id, username, deleted
FROM user
WHERE id = 1;
-- deleted=0 未删除, deleted=1 已删除

-- 3. 校验订单状态流转
SELECT id, order_no, status, amount, create_time
FROM orders
WHERE user_id = 1
ORDER BY create_time DESC;

-- 4. 校验学习记录
SELECT sr.id, sr.content, sr.duration, sr.type, sr.status, u.nickname
FROM study_record sr
JOIN user u ON sr.user_id = u.id
WHERE u.username = 'test001'
ORDER BY sr.create_time DESC;

-- 5. 校验技能数据
SELECT name, category, level, description
FROM skill
ORDER BY category, level;
```
