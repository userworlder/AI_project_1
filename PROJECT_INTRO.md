# 灵思·AI学伴 — 项目介绍

> 本文档面向测试人员，帮助了解项目全貌、功能模块、技术架构及测试环境搭建。

---

## 一、项目概述

**灵思·AI学伴** 是一款面向大学生的智能学习辅助系统，结合 AI 大语言模型技术，为学生提供智能问答、技能评估、模拟面试、学习追踪等全方位学习辅助服务。

项目采用**前后端分离架构**，包含三个核心子项目：

| 子项目 | 技术栈 | 说明 |
|-------|--------|------|
| **后端服务** | Java 17 + Spring Boot 3.2 + MySQL | API 服务与业务逻辑 |
| **管理后台** | Vue 3 + Element Plus | 管理员数据管理与运营 |
| **移动端** | HarmonyOS NEXT + ArkTS | 学生端 App |

## 二、功能模块

### 2.1 AI 聊天助手
- 基于大模型的智能问答与学习辅导
- 支持多轮对话，保持上下文连续
- Dify 平台接入，灵活切换 AI 模型
- Markdown 富文本渲染（加粗、代码、换行等）

### 2.2 技能评估
- AI 自动出题，覆盖多种技术栈
- 自动评分与智能反馈
- 技能掌握度可视化分析

### 2.3 模拟面试
- AI 驱动的技术面试模拟
- 真实面试场景还原
- 面试后智能复盘分析
- 个性化改进建议

### 2.4 简历评估
- 上传简历内容进行 AI 评估
- 多维度简历评分（技能匹配度、项目经验、排版等）
- 改进建议与优化方向

### 2.5 学习记录
- 跟踪用户学习进度
- 记录每次练习与评测结果
- 用户成长曲线可视化

### 2.6 管理后台（admin）
| 页面 | 功能 |
|------|------|
| 仪表盘 | 数据概览（用户数、学习时长、AI交互次数） |
| 用户管理 | 用户列表、搜索、新增/编辑/删除 |
| 技能树管理 | 技能 CRUD、分类管理、树形展示 |
| 学习记录 | 查看所有学习记录 |
| AI功能管理 | AI 配置管理（API Key、模型等） |
| 订单管理 | 订单列表、状态筛选、搜索、删除 |

```
                     │ HTTP/JSON + JWT Token
┌────────────────────┴────────────────────────────────────┐
│                    Nginx（可选）                         │
└────────────────────┬────────────────────────────────────┘
                     │
┌────────────────────┴────────────────────────────────────┐
│  后端服务（Spring Boot 3.2）                             │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐                │
│  │Controller│ │ Service  │ │  Mapper  │                │
│  │  层      │ │  层      │ │  层      │                │
│  └──────────┘ └──────────┘ └──────────┘                │
│  ┌──────────┐ ┌──────────┐ ┌──────────┐                │
│  │  Spring   │ │  JWT     │ │  Swagger │                │
│  │ Security  │ │  鉴权    │ │  API文档  │                │
│  └──────────┘ └──────────┘ └──────────┘                │
└──────────────┬──────────────────────────────────────────┘
               │
┌──────────────┴──────────────┐ ┌────────────────────────┐
│    MySQL 8.0               │ │    Dify AI 平台         │
│    数据库                  │ │    AI 能力提供          │
└─────────────────────────────┘ └────────────────────────┘

┌─────────────────────────────────────────────────────────┐
│               管理后台（Vue 3 + Element Plus）            │
│     Element UI · ECharts · Axios · Pinia                │
└─────────────────────────────────────────────────────────┘
```

### 3.2 后端分层架构
```
backend/src/main/java/com/aicompanion/
├── controller/    # 控制器层（接收 HTTP 请求）
├── service/       # 业务逻辑层
├── mapper/        # 数据访问层（MyBatis-Plus）
├── model/         # 数据模型
│   ├── entity/    # 实体类（对应数据库表）
│   ├── dto/       # 数据传输对象（接收参数）
│   └── vo/        # 视图对象（返回数据）
├── common/        # 公共模块
│   ├── response/  # 统一响应
│   ├── exception/ # 异常处理
│   └── util/      # 工具类
└── config/        # 配置类
```

### 3.3 数据库

- **数据库名：** `ai_companion`
- **ORM 框架：** MyBatis-Plus
- **主要表结构：**
  - `user` — 用户表
  - `skill` — 技能表
  - `study_record` — 学习记录表
  - `resume` — 简历评估表
  - `order` — 订单表
  - `ai_config` — AI 配置表

## 八、常见问题

### Q1: 后端启动报数据库连接失败？
检查 MySQL 是否启动，以及 `application.yml` 中的连接信息（地址、用户名、密码）是否正确。

### Q2: 登录后提示 401？
Token 已过期或无效，重新登录获取新的 Token。

### Q3: 管理后台页面显示空白？
确认后端服务已启动，或开启了模拟登录模式（localStorage 中有 mock 标识）。

### Q4: 移动端连不上后端？
- 模拟器使用 `http://10.0.2.2:8080`
- 真机使用电脑的局域网 IP 地址

### Q5: AI 聊天没有回复？
- 检查 Dify 服务是否运行
- 检查 DifyApi.ets 中的 API Key 配置
- 检查网络地址是否配置正确
