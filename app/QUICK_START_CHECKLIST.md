# 🚀 AI 聊天功能 - 快速启动检查清单

## ✅ 使用前必做事项

### 第一步：配置 Dify API（必须）

- [ ] **获取 API Key**
  - 访问你的 Dify 平台
  - 创建应用或进入现有应用
  - 复制 API Key（格式：`app-xxxxxxxxxxxxxxxxxxxx`）

- [ ] **修改配置文件**
  - 打开文件：`entry/src/main/ets/http/DifyApi.ets`
  - 找到第 11 行
  - 将 `'app-xxx'` 替换为你的实际 API Key
  
  ```typescript
  private static readonly API_KEY: string = 'app-你的实际APIKey'
  ```

- [ ] **确认网络地址**
  - 模拟器：使用 `http://10.0.2.2/v1`（默认已配置）
  - 真机：将 `10.0.2.2` 替换为电脑局域网 IP

---

### 第二步：确保 Dify 服务运行（必须）

- [ ] **检查 Dify 是否启动**
  ```bash
  # Docker 部署
  docker ps | grep dify
  
  # 应该看到 dify-api、dify-worker 等容器
  ```

- [ ] **测试 Dify API 可访问**
  ```bash
  # 在浏览器或命令行访问
  curl http://localhost/v1/chat-messages \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer app-你的APIKey" \
    -d '{"query":"你好","conversation_id":"","response_mode":"blocking","user":"test"}'
  ```
  
  **预期结果：** 返回 JSON 响应，包含 `answer` 字段

---

### 第三步：编译运行应用（必须）

- [ ] **打开 DevEco Studio**
  - 加载项目：`F:\Ai_project\project_1\app`

- [ ] **检查依赖**
  - 等待依赖下载完成
  - 确认无红色错误提示

- [ ] **编译项目**
  ```bash
  # 方式 1：DevEco Studio
  # 点击工具栏的 Run 按钮（绿色三角形）
  
  # 方式 2：命令行
  hvigorw assembleHap
  ```

- [ ] **安装到设备/模拟器**
  - 选择目标设备
  - 点击 Run
  - 等待安装完成

---

### 第四步：测试聊天功能（推荐）

- [ ] **进入聊天页面**
  - 启动应用后，在 Home 页面
  - 点击"AI智能对话"按钮

- [ ] **发送第一条消息**
  - 在输入框输入："你好"
  - 点击"发送"按钮
  
  **预期结果：**
  - ✅ 出现蓝色用户消息气泡（右对齐）
  - ✅ 等待片刻后，出现灰色 AI 回复气泡（左对齐）
  - ✅ 消息下方显示时间

- [ ] **测试多轮对话（练习一）**
  - 发送："我叫小明"
  - 等待回复
  - 发送："我叫什么名字？"
  
  **预期结果：**
  - ✅ AI 回复中包含"小明"
  - ✅ 说明上下文连续生效

- [ ] **测试 Markdown 渲染（练习二）**
  - 发送："请用 Markdown 格式介绍你自己，包含加粗和代码"
  
  **预期结果：**
  - ✅ AI 回复中的 `**加粗文本**` 显示为粗体
  - ✅ AI 回复中的 `` `代码` `` 显示为代码样式（等宽字体、浅灰背景）

- [ ] **测试清空功能**
  - 点击右上角"清空"按钮
  
  **预期结果：**
  - ✅ 消息列表清空
  - ✅ 显示空状态界面

## 🔍 问题排查

### 如果无法发送消息

**检查项：**
- [ ] Dify 服务是否运行？
- [ ] API Key 是否正确？
- [ ] 网络地址是否正确（模拟器 vs 真机）？
- [ ] 防火墙是否阻止连接？

**查看日志：**
```
DevEco Studio → Log 窗口 → 过滤 "[DifyApi]"
```

---

### 如果 Markdown 不渲染

**检查项：**
- [ ] AI 回复是否包含 Markdown 语法（`**text**` 或 `` `code` ``）？
- [ ] 是否在 Chat.ets 中使用了 `MarkdownRenderer` 组件？

**验证方法：**
查看 AI 回复原始内容，确认包含特殊符号。

---

### 如果多轮对话不连续

**检查项：**
- [ ] 第一次请求后，`conversationId` 是否保存？
- [ ] 第二次请求是否携带了 `conversationId`？

**查看日志：**
```
[DifyApi] 请求体: {...,"conversation_id":"",...}     // 第一次
[DifyApi] 响应体: {...,"conversation_id":"abc-123",...}
[DifyApi] 请求体: {...,"conversation_id":"abc-123",...}  // 第二次
```
